# 上下文增强 (Context Enhancement)

Qwen3-ASR 支持通过提供上下文（Context），对特定领域的专有词汇（如人名、地名、产品术语）进行识别优化。此功能比传统热词更灵活强大。

## 适用场景

- **专有名词识别**：优化人名、公司名、专业术语的识别。
- **特定领域优化**：如医疗、法律、金融等领域的长文本背景。

## 使用方法

在调用 API 时，通过 `messages` 中的 `system` 角色传入 `text` 内容。

### 请求结构示例

```json
{
    "model": "qwen3-asr-flash",
    "input": {
        "messages": [
            {
                "role": "system",
                "content": [
                    {
                        "text": "这里放入您的上下文内容，可以是词表、段落或混合文本。"
                    }
                ]
            },
            {
                "role": "user",
                "content": [
                    {"audio": "..."}
                ]
            }
        ]
    }
}
```

## 上下文内容格式

支持 **10000 Token** 以内的文本。

1.  **词表形式**
    ```text
    Bulge Bracket、Boutique、Middle Market、国内券商
    ```

2.  **自然语言段落**
    ```text
    最近有不少澳洲的小伙伴问我，到底什么是投行？... Bulge Bracket投行：这就是我们常说的九大投行...
    ```

3.  **混合内容** (词表 + 文本)
    ```text
    相关术语：ASR, NLP, TTS
    本文介绍语音识别技术的发展...
    ```

## 效果对比

| 场景 | 不使用上下文 | 使用上下文 |
| :--- | :--- | :--- |
| **投行术语** | "Bird Rock" | "Bulge Bracket" |
| **人名** | "王昊宣" | "王皓轩" (如果在上下文中提供了正确人名) |

## 注意事项

- 上下文内容无需完全精准匹配音频内容，模型具备鲁棒性，能从提供的背景信息中提取有效线索。
- 即使包含无关文本（如杂乱的人名列表），通常也不会对识别造成负面影响。
